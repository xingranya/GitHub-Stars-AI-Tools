# 项目分析：GitHub-Stars-AI-Tools

## 目标

重写一个与 `better-github-stars-manager` 同类的 GitHub Star 管理产品，但不继承浏览器插件形态。产品主线是：

- 自动同步用户全部 GitHub Stars。
- 保留 Star 管理核心能力：列表、搜索、筛选、标签、笔记、批量整理、增量/全量同步。
- 自动抓取仓库 README，生成中文摘要与中文译文。
- 将 Star 列表构建为个人 AI 知识库。
- 支持自然语言检索，让用户用需求描述找回已 Star 项目。

## 当前仓库状态

当前目录主要包含 `skills/` 规划与工程方法材料，暂未发现应用源码、依赖配置或既有运行入口。因此本轮工作按“从零规划新产品”处理，不进行源码迁移。

## 参考工具核心能力

来自 `better-github-stars-manager` 的 README 能力拆解：

| 能力 | 是否保留 | 新产品实现方式 |
| --- | --- | --- |
| Star 全量加载 | 保留 | 应用内通过 GitHub API 同步所有 starred repositories |
| 快速搜索和过滤 | 保留并增强 | 关键词检索 + 元数据过滤 + 语义检索 |
| 自定义标签 | 保留 | 本地注解层管理 tags |
| 笔记 | 保留 | 本地注解层管理 notes |
| 自动建议标签 | 保留并增强 | 基于 topics、language、README 摘要和 AI 分类推荐 |
| 增量同步和全量重扫 | 保留 | 同步状态机 + GitHub API 对账 |
| Gist 注解同步 | 保留为兼容方案 | 后续作为导入/导出或跨端同步选项 |
| GitHub 页面浮动按钮 | 不保留 | 独立应用不依赖 GitHub 页面入口 |
| Repo 页面 tag chip | 不进入主线 | 未来可作为伴侣扩展，不属于 MVP |

## 推荐首发形态

首发建议采用 `Tauri 2` 桌面应用：

- 更适合本地优先、私有数据、低成本 AI 缓存。
- 可以安全保存 GitHub Token 和 AI Provider Key。
- 可以后台执行同步、README 抓取、摘要翻译和向量化任务。
- 后续仍可复用前端与领域模块扩展 Web App。

## 成功标准

MVP 完成后，用户应能：

1. 连接 GitHub 账号。
2. 同步全部 Star 项目。
3. 在应用中搜索、筛选、打标签、写笔记。
4. 打开任意项目详情，看到中文摘要与 README 原文。
5. 通过中文自然语言描述找到相关 Star 项目，并看到匹配理由。

## 明确不做

- 不做 Chrome 插件。
- 不做 GitHub 页面 DOM 注入。
- 不做 Manifest V3 适配。
- 不把浏览器插件的 UI 形态作为兼容目标。
