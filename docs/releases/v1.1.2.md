# GitHub-Stars-AI-Tools v1.1.2

v1.1.2 聚焦 AI 配置可见性、本地模型兼容、仓库清单分享和近期反馈修复。这个版本让本地 OpenAI 兼容服务更容易接入，也让用户可以把整理好的 Stars 仓库清单、标签、笔记和阅读状态完整分享给别人。

## 新增功能

- 新增“导出所有仓库”：可将仓库清单、标签、笔记和阅读状态导出为私密 GitHub Gist。
- 新增“导入所有仓库”：从 Gist 恢复别人分享的仓库清单和整理结果，缺失仓库会自动创建为当前账号的本地记录。
- 导入仓库时按 `fullName` 匹配已有记录，后续真实 GitHub Stars 同步遇到同名仓库会合并到同一条本地记录，避免重复。
- 设置页数据备份区拆分为“导出注解”和“导出所有仓库”，并提供对应的“导入注解”和“导入所有仓库”入口。
- AI 配置测试改为弹窗流程，点击测试后立即显示当前服务、请求地址、模型 ID、测试阶段和最终结果。
- AI 配置测试改用流式请求，弹窗会实时展示模型输出片段、首字响应、总耗时、输出 token 和估算 token/s。

## AI 与本地模型

- OpenAI 兼容接口支持本机和局域网 HTTP 地址，适配 Ollama、LM Studio、Docker 内服务和局域网模型网关。
- 允许的 HTTP 地址包括 `localhost`、`127.0.0.1`、`0.0.0.0`、`[::1]`、`host.docker.internal`、`10.*`、`172.16-31.*` 和 `192.168.*`。
- 继续拒绝公网 HTTP 地址和带 userinfo 的伪装地址，降低误配风险。
- 本机 OpenAI 兼容服务仍支持不填写 API Key；局域网 HTTP 服务仍要求 API Key，避免无鉴权服务被误暴露。
- AI 测试失败时展示后端返回的友好错误，成功时展示模型名和摘要片段。

## 体验优化

- 概览页完成视觉重构，侧栏入口从“仪表盘”调整为“概览”。
- 概览页新增收藏仓库和 AI 用量走势线，README 覆盖率和笔记覆盖率进度条。
- 最近收藏支持语言筛选，并提供更清晰的仓库行操作。
- 同步、README 抓取、AI 摘要和标签网络任务进行时，概览页统计会随任务进度刷新。
- README 摘要对长内容截断提供更清晰提示。
- AI 标签网络降低单批处理仓库数，减少 OpenAI 兼容接口超时概率。

## 修复问题

- 修复点击“测试连接”后只显示按钮“测试中”，无法判断请求阶段和结果的问题。
- 修复 OpenAI 兼容模型返回非标准 JSON 字段别名时，批量 AI 摘要可能失败的问题。
- 修复本地或局域网模型 HTTP 地址因强制 HTTPS 校验而无法配置的问题。
- 修复仓库导入后再次同步同名 GitHub Stars 可能产生冲突或重复记录的问题。
- 修复后台任务完成后概览页统计更新不及时的问题。

## 数据与安全

- “导出所有仓库”不包含 README、AI 摘要、Embedding、API Key 或本机设置。
- GitHub Token 和 AI API Key 继续保存到系统凭据管理器，不写入导出文件。
- Gist 备份继续沿用 GitHub Token 的 `gist` 权限和私密 Gist 承载方式。
- 注解导入仍只恢复本地已有仓库的整理结果；完整仓库分享请使用“导入所有仓库”。
