# GitHub-Stars-AI-Tools v1.2.0

v1.2.0 将 GSAT 从“整理已有 Stars”进一步扩展为“发现、比较并收藏新项目”的本地优先知识库。本次更新新增 AI 同类项目推荐、GitHub 开源榜单和个人 Stars 榜单，同时完善 Stars 全量同步、仓库工作区布局和应用内更新体验。

## 发现与推荐

- 新增独立“发现”页面，可集中查看、筛选和管理 GitHub 项目推荐候选。
- 支持在仓库页选择 1 至 8 个参考项目，由 AI 分析项目特征、生成 GitHub 搜索式并查找同类项目。
- 推荐候选支持按待处理、已标记、已忽略、已收藏状态和项目类型筛选，并支持分页浏览。
- 支持将候选标记为稍后研究、暂时忽略、恢复为待处理，或直接加入个人 GitHub Stars。
- 推荐详情可查看并刷新项目 README；配置 AI 服务后，可生成并缓存 README 中文翻译。
- 部分 GitHub 搜索式失败时会保留其他成功候选，并在页面中明确提示未完成的搜索。
- 可查看最近使用的 GitHub 搜索式，便于了解推荐结果的来源。

## 排行榜

- 新增独立“排行榜”入口，包含“开源榜单”和“我的 Stars”两个工作区。
- 开源榜单提供趋势榜、新锐榜和热门榜，并支持按主流编程语言筛选、分页浏览和手动刷新。
- 我的 Stars 支持按仓库 Stars 数、最近更新时间和最近收藏时间排序。
- 榜单项目可打开详情查看 README；未收藏的开源项目可直接加入 GitHub Stars。
- GitHub 开源榜单结果缓存 6 小时；网络不可用时会回退到最近一次缓存并标记缓存时间。
- 榜单名称和排序规则均基于 GitHub 公开搜索结果，用于项目发现，不代表精确的 Stars 增长统计。

## Stars 同步

- Stars 同步改为完整分页拉取，不再只依赖单页结果，可覆盖账号下的全部收藏仓库。
- 只有全部分页成功后才进行权威对账，将 GitHub 上已取消收藏的仓库标记为已移除，避免部分请求失败造成误判。
- 同步过程会实时显示当前页和已扫描数量；完成后汇总当前、新增、更新、移除和扫描数量。
- 已重新收藏的仓库会恢复为有效状态，本地标签、笔记和其他整理结果继续保留。

## 仓库工作区

- 仓库列表新增同类项目多选模式，可批量选择推荐参考仓库，并在生成完成后直接进入发现页。
- 仓库列表和 AI 项目知识卡支持拖动调整宽度，布局宽度会在本机保存。
- 新增“发现”和“排行榜”导航入口，仓库管理、项目发现与榜单浏览之间的跳转更直接。
- 推荐和榜单详情延续现有 README Markdown 阅读体验，长内容可在独立详情区域滚动查看。

## 关于与应用更新

- 设置页新增独立“关于项目”页面，集中展示项目定位、源码仓库、问题反馈、许可证和第三方致谢。
- 应用更新说明改为 Markdown 渲染，支持标题、列表、代码、表格、链接和长内容滚动。
- 启动时发现新版本后，“查看更新”会直接打开更新弹窗，无需再跳转到设置页查找入口。
- 更新弹窗统一展示当前版本、可用版本、发布时间、下载进度和安装状态。
- 加强检查、下载、安装和重启流程的重复操作保护；安装完成后自动尝试重启，失败时提供手动重启或重新打开应用的恢复提示。

## 使用说明

- 发现推荐需要已连接的 GitHub 账号和可用的 AI 配置。
- 开源榜单需要 GitHub Token 获取公开搜索结果；个人榜单读取已同步到本机的 Stars 数据。
- README 中文翻译会使用当前 AI 配置，翻译结果按 README 内容缓存，内容更新后可重新生成。
