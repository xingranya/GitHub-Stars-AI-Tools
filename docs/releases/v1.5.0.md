# GitHub-Stars-AI-Tools v1.5.0

v1.5.0 为 GSAT 带来完整的本地向量检索能力。用户无需部署额外服务或填写 Embedding API Key，只需打开“向量检索”开关，即可在本机完成模型下载、仓库向量生成和语义索引构建。准备完成后，中英文自然语言检索可完全离线运行。

## 一键启用本地向量检索

- 向量检索默认使用内置的 `multilingual-e5-small` 中英双语模型。
- 首次启用时会明确提示约 490 MB 下载大小，用户确认后才开始下载。
- 设置页会依次展示下载、完整性校验、模型加载和仓库索引构建状态。
- 模型不会打进安装包，只保存在当前设备的应用缓存中。
- 首次准备完成后，模型加载、查询向量生成和 zvec 检索均可断网运行。
- 关闭开关后会立即恢复关键词检索，同时保留已下载模型和现有索引，方便以后快速重新启用。
- 关闭功能后可从设置页删除本地模型；删除模型不会清理 Stars、README、标签、笔记或本地数据库。

## 中英双语语义搜索

- 支持中文查中文、英文查英文、中文查英文和英文查中文。
- 查询和仓库知识文本采用 E5 推荐的查询/文档协议，并对向量结果进行归一化和相关度校准。
- 默认最低相关度调整为 `0.80`，减少语义相近但用途不符的结果。
- 40 条中英质量集实测：Recall@8 为 `0.925`，跨语言 Recall@8 为 `0.850`，MRR@8 为 `0.758`。
- 搜索结果继续保持最多 10 条；问候语不会触发仓库召回，短技术缩写仍优先进行精确匹配。
- 本地模型或索引暂时不可用时，搜索会自动降级为严格关键词检索，不阻断正常使用。

## 自动建库与增量维护

- 本地模型按批次生成仓库向量，减少大量 Stars 首次建库时的重复开销。
- Stars 元数据、README、AI 摘要、关键词或标签发生变化后，应用会自动更新对应仓库向量。
- 应用启动、相关后台任务完成和定时检查时都会处理待更新仓库，无需手动反复重建。
- SQLite 继续作为向量数据的可靠来源，zvec 作为可删除、可自动恢复的本地检索索引。
- zvec 索引使用临时快照和原子切换，构建失败不会暴露半完成索引。
- 索引损坏或与 SQLite 不一致时会自动恢复；部分仓库处理失败时会保留成功结果，并允许继续重试。

## 下载校验与故障恢复

- 模型固定到明确的 Hugging Face revision，避免上游文件变化导致模型与索引不一致。
- ONNX 模型、Tokenizer 和配置文件均校验文件大小与 SHA-256，全部通过后才允许加载。
- 下载中断、文件不完整或缓存损坏时会进入可重试状态，不会将损坏模型标记为可用。
- 重试时会清理已确认损坏的下载缓存，避免持续命中同一份错误文件。
- 模型准备采用全局单任务控制，并发启用、重试或搜索请求不会重复下载和加载同一模型。

## 设置与远程服务兼容

- 普通设置只保留向量检索开关、当前阶段、已索引仓库数、失败数和模型缓存占用。
- OpenAI 与 OpenAI 兼容 Embedding 保留在“高级远程 Embedding 设置”中。
- 本地模式固定使用官方模型、384 维向量和内置文本协议，前端配置无法覆盖这些关键参数。
- 旧版远程 Embedding 配置保持原有 Provider、地址、模型和凭据，不会因升级被删除。
- 旧版关闭状态会迁移为“本地模式但保持关闭”，不会自动下载模型。

## 数据与隐私

- 本地模式不会把仓库知识文本、查询内容或向量发送到远程服务。
- 个人笔记不会进入 Embedding 文本，也不会因启用本地向量检索改变原有隐私边界。
- 模型缓存位于应用缓存目录；SQLite、用户注解和 zvec 索引继续位于应用数据目录。
- 模型或索引升级不会重置用户数据库，原有 Stars、标签、笔记、README 和 AI 文档继续保留。
- 只有用户主动选择高级远程 Embedding 时，仓库知识文本才会发送到对应服务商。

## 兼容性与发布平台

- 桌面应用版本统一升级为 `1.5.0`。
- 正式支持 macOS Apple Silicon、Windows x64 和 Linux x64。
- 从 v1.5.0 起停止提供 macOS Intel 安装包；Intel 用户可继续使用最后一个兼容版本。
- 本地 Embedding 使用 CPU 推理，不要求独立显卡或额外安装 Python、ONNX Runtime、zvec 服务。

## 使用说明

1. 连接 GitHub 账号并完成 Stars 同步。
2. 打开“设置 → AI 引擎配置 → 向量检索”。
3. 打开向量检索开关，并确认首次模型下载。
4. 等待状态显示“本地向量检索已就绪”。
5. 进入 AI 搜索，直接使用中文或英文描述想查找的项目能力。

首次建库耗时取决于网络速度、CPU 性能和 Stars 数量。下载完成后的日常查询不依赖网络；仓库知识发生变化时，应用会在后台自动补齐向量。
